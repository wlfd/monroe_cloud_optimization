**AnomalyCard sub-component (inline in same file):**

Each anomaly card shows:
- Left: colored severity dot (`w-3 h-3 rounded-full` with `severityDotColor[anomaly.severity]`)
- Center-left: service name (bold), severity Badge, status Badge
- Center: generated description (e.g. "Spend increased 156% in us-east-1"), resource group, detected timestamp (`format(new Date(anomaly.detected_date), 'MMM d, yyyy')` — use `date-fns` if available in the project, otherwise use `new Date(...).toLocaleDateString()`)
- Right: estimated impact in red (`+$X,XXX` formatted with `toLocaleString('en-US', { style: 'currency', currency: 'USD' })`), then 4 action buttons stacked or in a row

Action buttons:
- **Investigate** (outline button, small): calls `updateStatus.mutate({ id: anomaly.id, status: 'investigating' })`. Disabled if `anomaly.status === 'investigating'` or `'resolved'` or `'dismissed'`.
- **Dismiss** (outline button, small): calls `updateStatus.mutate({ id: anomaly.id, status: 'dismissed' })`. Disabled if `anomaly.status === 'dismissed'` or `'resolved'`.
- **Mark as Expected** (ghost button, small): calls `markExpected.mutate({ id: anomaly.id })`. Disabled if `anomaly.expected === true`.
- **View Resources** (ghost button, small, external link icon): renders as a non-functional placeholder for now (no resource detail page in scope). Show as disabled or greyed out — Claude's discretion.

Card layout: use `Card` with `CardContent` having `flex` layout. Keep the card visually scannable — severity dot on far left, impact on far right in red.

KPI Cards: use the same loading skeleton pattern as DashboardPage. Show `Skeleton` while `summary` is undefined. Display `summary?.active_count ?? 0`, `$${(summary?.total_potential_impact ?? 0).toLocaleString(...)`, `summary?.resolved_this_month ?? 0`, and `summary?.detection_accuracy != null ? summary.detection_accuracy.toFixed(1) + '%' : 'N/A'`.
