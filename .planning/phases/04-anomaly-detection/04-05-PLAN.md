If the anomalies table is empty, directly check the detection function works:

asyncio.run(test())

**Step 1 — Anomalies page loads (ANOMALY-04)**
1. Open browser to http://localhost:5173/anomalies
2. Verify: 4 KPI summary cards at the top (Active Anomalies, Potential Impact, Resolved This Month, Detection Accuracy)
3. Verify: At least some anomaly cards visible (or a clear empty state if seed data produces no anomalies above threshold)
4. Verify: Each card shows — colored dot (red/orange/blue), service name, severity badge, status badge, human-readable description, resource group, estimated impact in red

**Step 2 — Severity classification visible (ANOMALY-02)**
1. Check that anomaly cards show correct severity: cards with estimated impact ≥ $1,000 show "Critical" with red dot, $500-999 show "High" with orange dot, $100-499 show "Medium" with blue dot
2. Verify the section header shows severity counts (e.g. "1 Critical · 2 High · 3 Medium")

**Step 3 — Dollar impact displayed (ANOMALY-03)**
1. Verify "Estimated Impact" (the +$X,XXX figure) is shown prominently in red on each card
2. Verify the Potential Impact KPI card shows the sum of all active anomaly impacts

**Step 4 — Filter dropdowns work (ANOMALY-04)**
1. Open the Severity dropdown, select "Critical" — verify only Critical anomalies appear (or empty state if none)
2. Select "All" — verify all anomalies return
3. Try Service and Resource Group filters if values are available

**Step 5 — Lifecycle actions work (ANOMALY-04)**
1. Find a "New" status anomaly card. Click **Investigate** — verify the status badge changes to "Investigating" without page reload
2. Find another anomaly. Click **Dismiss** — verify the status badge changes to "Dismissed"
3. Find another anomaly. Click **Mark as Expected** — verify it changes to "Dismissed" (or disappears from active view)

**Step 6 — Export Report (ANOMALY-04)**
1. Click the "Export Report" button in the page header
2. Verify a file `anomaly-report.csv` downloads
3. Open the file — verify it has header row and data rows with detected_date, service_name, severity, estimated_monthly_impact columns

**Step 7 — Dashboard integration (ANOMALY-04)**
1. Navigate to http://localhost:5173/ (Dashboard)

4. Click "View anomalies →" link — verify it navigates to /anomalies

**Step 8 — Detection config panel visible**
1. Scroll to the bottom of /anomalies
