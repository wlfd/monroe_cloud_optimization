  <what-built>
Complete Phase 6 multi-tenant attribution feature:
- Three new database tables (tenant_profiles, allocation_rules, tenant_attributions)
- Daily attribution job triggered after each ingestion run (maps billing_records.tag to tenants)
- 4 attribution API endpoints + 8 settings API endpoints

- Settings page: tabbed tenant name management + allocation rule CRUD
  </what-built>
  <how-to-verify>
Ensure the application is running (docker compose up). Log in at http://localhost:5173.

**Step 1 — ATTR-01: Verify automatic tenant mapping**
1. Trigger an ingestion run: navigate to Admin > Ingestion, click "Run Now", wait for completion
2. Navigate to Attribution in the sidebar
3. Confirm the page loads without errors and shows a table (may be empty if no billing data, or show tenant rows if mock data includes tenant_id tags)
4. If using MOCK_AZURE=true: check that any mock billing records with tenant_id tags produced TenantProfile rows by visiting Settings > Tenants — tenant IDs should appear

**Step 2 — ATTR-02: Verify allocation rule management**
1. Navigate to Settings (sidebar)
2. Confirm Settings page has two tabs: "Tenants" and "Allocation Rules"
3. Click "Allocation Rules" tab
4. Click "Add Rule" — confirm an inline row appears (NOT a modal)
5. Fill in: Target Type = resource_group, Target Value = "rg-shared", Method = by_count
6. Click Save — confirm the rule appears in the rules table with Priority 1
7. Add a second rule — confirm it gets Priority 2
8. Change the Priority of rule 2 to 1 and blur — confirm rules reorder to maintain unique priorities
9. Delete a rule — confirm it disappears and remaining rules renumber

**Step 3 — ATTR-03: Verify per-tenant attribution view**
1. Navigate to Attribution
2. Confirm the table shows columns: Tenant, Monthly Cost, % of Total, MoM Change, Top Category
3. Confirm the month/year selectors default to the current month and year
4. Click a previous month — confirm the table updates (empty or different data)
5. Click a tenant row — confirm it expands inline to show a service-category cost breakdown
6. Click the same row again — confirm it collapses
7. If any tenant has is_new=true: confirm a "New" badge appears on that row
8. Navigate to Settings > Tenants — click "Acknowledge" next to a New tenant — confirm the badge disappears in Attribution
9. If UNALLOCATED costs exist: confirm an UNALLOCATED row appears at the bottom of the attribution table

**Step 4 — ATTR-04: Verify CSV export**
1. Navigate to Attribution (any month with data)
2. Click "Export CSV"
3. Confirm a file named `attribution-YYYY-MM.csv` downloads
4. Open the file — confirm it contains a header row: tenant_id, display_name, period, total_cost_usd, pct_of_total, mom_delta_usd, top_service_category
5. Confirm data rows have correct values (match what was shown in the UI table)

**Step 5 — Spot checks**
- Settings > Tenants: click a display name, edit it, save — confirm the attribution table shows the updated name
- Attribution sorting: click column headers — confirm ascending/descending sort works
- All navigation links work: Dashboard, Anomalies, Recommendations, Attribution, Settings
