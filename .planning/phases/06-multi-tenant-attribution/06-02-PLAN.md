class TenantDisplayNameUpdate(BaseModel):
    display_name: str

# AllocationRuleResponse — GET /settings/rules
class AllocationRuleResponse(BaseModel):
    id: uuid.UUID
    priority: int
    target_type: str   # 'resource_group' | 'service_category'
    target_value: str
    method: str        # 'by_count' | 'by_usage' | 'manual_pct'
    manual_pct: dict[str, float] | None
    created_at: datetime
    model_config = ConfigDict(from_attributes=True)

# AllocationRuleCreate — POST /settings/rules
class AllocationRuleCreate(BaseModel):
    target_type: str
    target_value: str
    method: str
    manual_pct: dict[str, float] | None = None

    @model_validator(mode="after")
    def validate_manual_pct(self):
        if self.method == "manual_pct":
            if not self.manual_pct:
                raise ValueError("manual_pct is required when method='manual_pct'")
            total = sum(self.manual_pct.values())
            if abs(total - 100.0) > 0.01:
                raise ValueError(f"manual_pct values must sum to 100 (got {total:.2f})")
        return self

# AllocationRuleUpdate — PATCH /settings/rules/{rule_id} (same fields, all optional)
class AllocationRuleUpdate(BaseModel):
    target_type: str | None = None
    target_value: str | None = None
    method: str | None = None
    manual_pct: dict[str, float] | None = None

# RuleReorderRequest — POST /settings/rules/reorder
class RuleReorderRequest(BaseModel):
    rule_ids: list[uuid.UUID]  # ordered list; service renumbers 1..N sequentially

# TenantAttributionResponse — GET /attribution/
class TenantAttributionResponse(BaseModel):
    tenant_id: str
    display_name: str | None
    year: int
    month: int
    total_cost: float
    pct_of_total: float
    mom_delta_usd: float | None
    top_service_category: str | None
    allocated_cost: float
    tagged_cost: float
    computed_at: datetime
    model_config = ConfigDict(from_attributes=True)

# ServiceBreakdownItem — GET /attribution/breakdown/{tenant_id}

    total_cost: float

**Function: `run_attribution()`**

Main daily job. Steps:
1. Open `AsyncSessionLocal` session
2. Determine current year/month (from MAX(usage_date) in billing_records — same pattern as anomaly detection's `check_date` approach for robustness)
3. **Discover tenants:** `SELECT DISTINCT tag FROM billing_records WHERE tag != ''` → list of tenant_ids. For each: UPSERT into `tenant_profiles` with `on_conflict_do_update` on `tenant_id`. On INSERT set `is_new=True`, `first_seen=today`. On conflict, leave `is_new` unchanged (don't reset acknowledged tenants). Use `pg_insert` upsert pattern from ingestion.py.
4. **Compute tagged costs:** `SELECT tag, SUM(pre_tax_cost) FROM billing_records WHERE year=Y AND month=M AND tag != '' GROUP BY tag` → `{tenant_id: Decimal}` dict.
5. **Compute untagged costs:** `SELECT SUM(pre_tax_cost) FROM billing_records WHERE year=Y AND month=M AND tag = ''` → single Decimal.
6. **Load rules ordered by priority:** `SELECT * FROM allocation_rules ORDER BY priority ASC`.
7. **Apply allocation rules** to untagged_cost:
   - For each rule in priority order, compute a subset of `untagged_cost` matching the rule's `target_type`/`target_value`:
     - `resource_group` → `WHERE resource_group = target_value AND tag = ''`
     - `service_category` → `WHERE service_name = target_value AND tag = ''`
   - Apply `apply_allocation_rule(rule_cost, rule.method, rule.manual_pct, tenant_tagged_costs)` to get `{tenant_id: amount}` allocations
   - Track which tenant_id gets how much allocated cost (accumulate `allocated_per_tenant` dict)
   - Sum all matched rule costs → `total_rule_matched`. `unallocated = untagged_cost - total_rule_matched`
8. **Compute totals per tenant:** `total_per_tenant = tagged_costs + allocated_per_tenant` (merge dicts). Add 'UNALLOCATED' key with `unallocated` amount (if > 0).
9. **Compute grand total:** sum all values including UNALLOCATED.
10. **Fetch prior month totals** for MoM delta: query `tenant_attributions` for `(year=prior_Y, month=prior_M)` → dict. Compute delta as `current - prior` (None if not present).
11. **Upsert TenantAttribution rows** via `pg_insert(...).on_conflict_do_update(index_elements=["tenant_id", "year", "month"], set_={...})`. Upsert one row per tenant including UNALLOCATED.
12. Commit and close session.

**Helper: `apply_allocation_rule(cost, method, manual_pct, tenant_costs) -> dict[str, float]`**

Implement the three methods:
- `by_count`: `cost / len(tenant_costs)` per tenant. If no tenants, return empty dict.
- `by_usage`: proportional by tenant's tagged cost. If `sum(tenant_costs.values()) == 0`, fall back to by_count.
- `manual_pct`: `{t: cost * (pct / 100.0) for t, pct in manual_pct.items()}`.

**Function: `get_attributions(session, year, month) -> list[TenantAttribution + display_name]`**

Query `tenant_attributions` for year/month. LEFT JOIN `tenant_profiles` on `tenant_id` to get `display_name`. Return rows ordered by `total_cost DESC`. Include UNALLOCATED row if it exists. Since TenantAttribution doesn't have a relationship to TenantProfile, do a two-step query: fetch attributions, fetch profiles as dict, attach display_name in Python.

**Function: `get_attribution_breakdown(session, tenant_id, year, month) -> list[{service_name, total_cost}]`**

Query `billing_records` on-the-fly (fresh data, not pre-computed):
```sql
SELECT service_name, SUM(pre_tax_cost) AS total_cost
FROM billing_records
WHERE tag = :tenant_id AND EXTRACT(YEAR FROM usage_date) = :year AND EXTRACT(MONTH FROM usage_date) = :month
GROUP BY service_name
ORDER BY total_cost DESC
```
For UNALLOCATED tenant_id, query `WHERE tag = ''` (untagged records).

**CRUD functions for settings:**
- `list_tenant_profiles(session)`: SELECT all from tenant_profiles ORDER BY first_seen.
- `update_tenant_display_name(session, tenant_id, display_name)`: UPDATE tenant_profiles SET display_name=? WHERE tenant_id=?.
- `acknowledge_tenant(session, tenant_id)`: UPDATE tenant_profiles SET is_new=False, acknowledged_at=utcnow() WHERE tenant_id=?.
- `list_allocation_rules(session)`: SELECT all ORDER BY priority ASC.
- `create_allocation_rule(session, rule_data: AllocationRuleCreate)`: INSERT. Assign `priority = MAX(priority) + 1` (or 1 if no rules exist).
- `update_allocation_rule(session, rule_id, rule_data: AllocationRuleUpdate)`: UPDATE only provided fields.
- `delete_allocation_rule(session, rule_id)`: DELETE then renumber remaining rules 1..N by current priority order.
- `reorder_allocation_rules(session, rule_ids: list[uuid.UUID])`: Renumber rules to match the provided order (1, 2, 3...) using the pg_insert upsert or simple UPDATE loop.
