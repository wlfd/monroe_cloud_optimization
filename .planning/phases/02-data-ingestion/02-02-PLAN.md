**`_fetch_page_sync(client, scope, query_def) -> tuple[list, list[str], str | None]`** — synchronous function (called via asyncio.to_thread). Calls `client.query.usage(scope=scope, parameters=query_def)`. Returns `(rows, column_names, next_link)`. If result is None, returns `([], [], None)`. Column names are extracted as `[c.name for c in result.columns]`.

**`_fetch_next_page_sync(token, next_url) -> tuple[list, list[str], str | None]`** — follows pagination using a raw httpx GET (NOT the SDK). Import httpx inside the function body. Use `token` (a bearer token string) to set the Authorization header. Parse the JSON response for `properties.rows`, `properties.columns`, and `properties.nextLink`. If httpx is not available, raise NotImplementedError with message "Install httpx for pagination support". Note: For MVP, the chunked-by-week approach in fetch_billing_data avoids pagination entirely — this function is a safety net.

**`async def fetch_billing_data(scope: str, start: datetime, end: datetime) -> list[dict]`** — main fetch function. If `settings.MOCK_AZURE` is True, return synthetic data: a list of 3 dicts with keys matching the real response (usage_date as a date int like 20260101, subscription_id from settings.AZURE_SUBSCRIPTION_ID, resource_group, service_name, meter_category, pre_tax_cost as float, currency="USD"). This allows local dev without Azure credentials.

If MOCK_AZURE is False: build client, build QueryDefinition with:
- type="Usage", timeframe="Custom"
- time_period=QueryTimePeriod(**{"from": start.isoformat(), "to": end.isoformat()})
- dataset=QueryDataset(granularity="Daily", aggregation={"totalCost": QueryAggregation(name="PreTaxCost", function="Sum")}, grouping=[SubscriptionId, ResourceGroup, ServiceName, MeterCategory — all DIMENSION type])

Call `await asyncio.to_thread(_fetch_page_sync, client, scope, query_def)`. Iterate rows, zip with columns, append to all_records. Do NOT follow next_link for MVP (chunked date windows keep pages small). Return all_records as list[dict].

**Module-level state (concurrency guard):**
```python
import asyncio
_ingestion_lock = asyncio.Lock()
_ingestion_running: bool = False

def is_ingestion_running() -> bool:
    return _ingestion_running
```

**`async def get_last_successful_run(session: AsyncSession) -> IngestionRun | None`** — SELECT FROM ingestion_runs WHERE status='success' ORDER BY started_at DESC LIMIT 1. Use `select(IngestionRun).where(IngestionRun.status == "success").order_by(IngestionRun.started_at.desc()).limit(1)`.

**`async def compute_delta_window(session: AsyncSession) -> tuple[datetime, datetime]`** — implements the locked decision: delta-only fetch from last successful run's window_end to now, capped at MAX_CATCHUP_DAYS=7. If no prior successful run exists, return (now - 4h, now) as the first scheduled run's window. The backfill is handled separately. Include a 24-hour overlap: set start = last_run.window_end - timedelta(hours=24) to catch late-arriving Azure records (Claude's discretion — resolving the open question from research in favor of the 24h re-check window, matching Pattern 6 in research).

**`async def upsert_billing_records(session: AsyncSession, records: list[dict]) -> int`** — uses PostgreSQL-specific `insert().on_conflict_do_update()`. Import: `from sqlalchemy.dialects.postgresql import insert as pg_insert`. Conflict target: index_elements=["usage_date", "subscription_id", "resource_group", "service_name", "meter_category"]. On conflict update: pre_tax_cost, currency, updated_at. If records is empty, return 0. Parse each raw dict from Azure API response — map column names to model fields. Azure returns `PreTaxCost` (float), `SubscriptionId`, `ResourceGroup`, `ServiceName`, `MeterCategory`, `UsageDate` (int like 20260131 = YYYYMMDD). Parse UsageDate int to Python date: `date(year=d//10000, month=(d%10000)//100, day=d%100)`. Return `result.rowcount`.

**`async def log_ingestion_run(session: AsyncSession, *, status: str, records_ingested: int = 0, error_detail: str | None = None, triggered_by: str, window_start: datetime | None = None, window_end: datetime | None = None, retry_count: int = 0) -> IngestionRun`** — creates and commits an IngestionRun row with completed_at=utcnow.

**`async def create_ingestion_alert(session: AsyncSession, *, error_detail: str, retry_count: int = 3) -> IngestionAlert`** — creates IngestionAlert row with is_active=True, error_message=error_detail, retry_count=retry_count, failed_at=utcnow. Commits.

**`async def clear_active_alerts(session: AsyncSession) -> None`** — UPDATE ingestion_alerts SET is_active=False, cleared_at=utcnow(), cleared_by='auto_success' WHERE is_active=True. Use `update(IngestionAlert).where(IngestionAlert.is_active == True).values(is_active=False, cleared_at=utcnow(), cleared_by="auto_success")`.

**`async def run_backfill(session: AsyncSession) -> None`** — chunked 24-month backfill. Check first if any successful run exists (use get_last_successful_run); if one exists, skip backfill (prevents re-triggering on restart). Otherwise: from now() going back 24 months, in 30-day chunks. For each chunk: call fetch_with_retry, then upsert_billing_records. Add `await asyncio.sleep(1)` between chunks (QPU quota: 12 QPU/10s). After all chunks complete, call log_ingestion_run with triggered_by="backfill", status="success".

**`async def _do_ingestion(triggered_by: str) -> None`** — the core orchestration:
```python
async with AsyncSessionLocal() as session:
    # Create a 'running' run record to track start
    run_start = datetime.now(timezone.utc)
    try:
        # First run? trigger backfill
        last_run = await get_last_successful_run(session)
        if last_run is None and triggered_by != "backfill":
            await run_backfill(session)
            return

        start, end = await compute_delta_window(session)
        scope = settings.AZURE_SUBSCRIPTION_SCOPE or f"/subscriptions/{settings.AZURE_SUBSCRIPTION_ID}"
        records = await fetch_with_retry(scope=scope, start=start, end=end)
        count = await upsert_billing_records(session, records)
        await clear_active_alerts(session)
        await log_ingestion_run(
            session, status="success", records_ingested=count,
            triggered_by=triggered_by, window_start=start, window_end=end
        )
    except Exception as exc:
        await log_ingestion_run(
            session, status="failed", error_detail=str(exc),
            triggered_by=triggered_by
        )
        await create_ingestion_alert(session, error_detail=str(exc), retry_count=3)
        raise
```

**`async def run_ingestion(triggered_by: str = "scheduler") -> None`** — concurrency guard wrapper:
```python
global _ingestion_running
if _ingestion_lock.locked():
    return
async with _ingestion_lock:
    _ingestion_running = True
    try:
        await _do_ingestion(triggered_by)
    finally:
        _ingestion_running = False
```

**`async def recover_stale_runs(session: AsyncSession) -> None`** — called on app startup. Finds ingestion_runs with status='running' and updates them to status='interrupted'. Prevents stale 'running' state after crash restart (Pitfall 5 from research).

Use `AsyncSessionLocal` directly (not the `get_db` dependency) — scheduler jobs run outside request context (Pattern 7 from research).
