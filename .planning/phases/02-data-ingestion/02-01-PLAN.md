Create `backend/app/models/billing.py` with three SQLAlchemy 2.0 mapped-column models following the exact pattern from `backend/app/models/user.py`:

**BillingRecord** (`__tablename__ = "billing_records"`):
- `id`: UUID primary key (uuid.uuid4 default)
- `usage_date`: Date, nullable=False
- `subscription_id`: String(255), nullable=False
- `resource_group`: String(255), nullable=False, default=""
- `service_name`: String(255), nullable=False, default=""
- `meter_category`: String(255), nullable=False, default=""
- `pre_tax_cost`: Numeric(18, 6), nullable=False
- `currency`: String(10), nullable=False, default="USD"
- `ingested_at`: DateTime(timezone=True), nullable=False, default=utcnow
- `updated_at`: DateTime(timezone=True), nullable=False, default=utcnow, onupdate=utcnow
- `__table_args__`: UniqueConstraint on ("usage_date", "subscription_id", "resource_group", "service_name", "meter_category", name="uq_billing_record_key") + four indexes: idx_billing_usage_date, idx_billing_subscription, idx_billing_resource_group, idx_billing_service_name

**IngestionRun** (`__tablename__ = "ingestion_runs"`):
- `id`: UUID primary key
- `started_at`: DateTime(timezone=True), nullable=False, default=utcnow
- `completed_at`: DateTime(timezone=True), nullable=True
- `status`: String(50), nullable=False — values: 'running' | 'success' | 'failed' | 'interrupted'
- `triggered_by`: String(50), nullable=False — values: 'scheduler' | 'manual' | 'backfill'
- `records_ingested`: Integer, nullable=False, default=0
- `window_start`: DateTime(timezone=True), nullable=True
- `window_end`: DateTime(timezone=True), nullable=True
- `retry_count`: Integer, nullable=False, default=0
- `error_detail`: Text, nullable=True
- Index on `started_at`: idx_ingestion_runs_started_at

**IngestionAlert** (`__tablename__ = "ingestion_alerts"`):
- `id`: UUID primary key
- `created_at`: DateTime(timezone=True), nullable=False, default=utcnow
- `error_message`: Text, nullable=False
- `retry_count`: Integer, nullable=False
- `failed_at`: DateTime(timezone=True), nullable=False
- `is_active`: Boolean, nullable=False, default=True
- `cleared_at`: DateTime(timezone=True), nullable=True
- `cleared_by`: String(50), nullable=True — 'auto_success' | 'admin'
- Index on `is_active`: idx_ingestion_alerts_active

Use the same utcnow helper from user.py (define it in billing.py as well — do not import from user.py).

Imports needed: uuid, datetime, timezone, date, Decimal (from decimal), String, DateTime, Date, Numeric, Integer, Boolean, Text, Index, UniqueConstraint (from sqlalchemy), UUID (from sqlalchemy.dialects.postgresql), Mapped, mapped_column (from sqlalchemy.orm), Base (from app.core.database).

---

**Step 2: Generate the Alembic migration**

From the `backend/` directory, run:
```bash
alembic revision --autogenerate -m "billing_ingestion_tables"
```

This produces a new file in `backend/migrations/versions/`. The migration MUST contain `op.create_table` calls for all three tables: `billing_records`, `ingestion_runs`, `ingestion_alerts`.

**CRITICAL — Verify the migration is not empty:** Open the generated file and confirm it contains at minimum:
- `op.create_table('billing_records', ...)` with all columns
- `op.create_table('ingestion_runs', ...)` with all columns
- `op.create_table('ingestion_alerts', ...)` with all columns
- `op.create_unique_constraint('uq_billing_record_key', 'billing_records', [...])`
- Multiple `op.create_index(...)` calls

If the migration is empty (no `op.create_table` calls), it means the billing import is missing from env.py — fix and regenerate.

**Step 3: Apply the migration**

```bash
alembic upgrade head
```

Must exit 0 with no errors.

**Pitfall from research:** Alembic only detects models imported into migrations/env.py before `target_metadata = Base.metadata`. The billing import must be present before that line.
