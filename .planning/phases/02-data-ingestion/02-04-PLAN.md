Create `frontend/src/pages/IngestionPage.tsx`.

**Layout structure:**
- Page heading: "Ingestion" with a subtitle "Azure billing data pipeline"
- Alert banner section (conditionally rendered)
- Status + Run Now row
- Run history table

**Alert banner (locked decision — INGEST-05):**
When `alerts` array has any entry with `is_active: true`, render a prominent banner above everything else:
```
[Red AlertTriangle icon]  Ingestion Failure
Error: {alert.error_message}
Retries attempted: {alert.retry_count} | Failed at: {formatted alert.failed_at}
```
Use shadcn Card with `border-red-500 bg-red-50 text-red-900` classes (or equivalent destructive variant). The banner persists until the next successful run clears it (auto-cleared on next poll). Do NOT add a dismiss button — the user decision specified auto-clear behavior on success.

**Status + Run Now row (locked decisions):**
- Left: Status badge — if `status.running` is true: green dot + "Running", else gray dot + "Idle"
- Right: "Run Now" Button — disabled when `status.running` is true (locked decision: block new triggers while running). On click: POST /api/v1/ingestion/run. On 202: show a brief success toast ("Ingestion started") and refresh runs list. On 409: show toast "Already running". Use try/catch with axios error handling.

**Run history table (locked decision — INGEST-06):**
shadcn Table (or a simple `<table>` with Tailwind) with columns:
- Timestamp (started_at formatted as local datetime)
- Status (badge: green "Success", red "Failed", yellow "Running", gray "Interrupted")
- Triggered By (pill: "Scheduled", "Manual", "Backfill")
- Records Ingested (right-aligned number)
- Duration (completed_at - started_at in seconds, e.g. "12s" or "—" if null)
- Error (truncated error_detail, max 60 chars, full text on hover title attribute, "—" if null)

Show last 20 runs (the API default). Show "No runs yet" empty state if array is empty.

**Data fetching:**
- On mount: fetch all three endpoints concurrently with `Promise.all([...])`:
  - GET /api/v1/ingestion/status
  - GET /api/v1/ingestion/runs?limit=20
  - GET /api/v1/ingestion/alerts?active_only=true
- Poll status every 5 seconds while mounted (`setInterval` in useEffect, clear on unmount). When status transitions from running→idle, also refresh runs and alerts.
- Loading state: show skeleton placeholder while initial fetch is in progress.
- Error state: if API returns 403, show "Admin access required" message.

**Imports to use:** `api` from `@/services/api`, shadcn components (Card, Button, Badge, Table if available — use whatever is already installed from Phase 1). Use lucide-react AlertTriangle for the banner icon. Use React hooks: useState, useEffect, useCallback, useRef (for interval ID).

**Admin guard:** Check user role from useAuth(). If user role is not 'admin', render "You don't have permission to view this page." instead of the full page content. This is a soft guard — the backend enforces 403.
